Issue 01 · Circular Import

도메인 분리 후 순환 참조가 터졌습니다.

AuthService와 BusinessService가 서로를 참조하면서 FastAPI Depends 런타임에서 import 문제가 발생했습니다.

해결생성자 주입을 제거하고 메서드 주입 + Router Orchestration으로 결합을 상위 레이어에서 조율했습니다.
Issue 02 · Rollback

성공 로그는 1,282건인데 DB에는 일부만 남았습니다.

중복 키 에러 처리 중 session.rollback()이 마지막 commit 이후 작업 전체를 되돌리는 문제가 있었습니다.

해결예외 처리 대신 PostgreSQL UPSERT를 적용해 중복을 정상 흐름으로 처리했습니다.
Issue 03 · File Reliability

파일명은 PDF인데 실제 파일은 달랐습니다.

기업마당 API 첨부파일의 확장자와 실제 바이너리 형식이 달라 파싱 실패와 전체 배치 롤백이 발생했습니다.

해결Magic Number 탐지와 SAVEPOINT 중첩 트랜잭션으로 실패 건만 제외하고 수집을 계속하도록 개선했습니다.
Issue 04 · Supabase Timeout

Full Sync가 200~300건 이후 멈췄습니다.

AI 구조화와 대량 저장을 단일 장수 세션으로 처리하며 Supabase pgbouncer timeout에 걸렸습니다.

해결페이지별 fresh session 구조, pool_pre_ping, pool_recycle, prepared statement cache 해제로 배치 안정성을 확보했습니다.
