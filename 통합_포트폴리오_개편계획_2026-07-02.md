# 한 HTML 안에 3개의 독립 포트폴리오를 넣는 개편 계획

작성일: 2026-07-02

## 1. 다시 잡은 기준

이번 작업은 페이지 수를 먼저 정해놓고 내용을 억지로 끼워 넣는 작업이 아니다.

정확한 기준은 이렇다.

- 에이전트 개발자용 포트폴리오는 이미 만들어둔 현재 포트폴리오를 유지한다.
- 백엔드 개발자용 포트폴리오는 `BIZ-FUND-AI`에서 쓸 수 있는 내용을 전부 꺼낸 뒤 페이지를 만든다.
- 솔루션엔지니어용 포트폴리오도 `BIZ-FUND-AI`에서 비즈니스, 문제정의, 사용자 여정, 운영 구조를 전부 꺼낸 뒤 페이지를 만든다.
- 최종 페이지 수는 먼저 정하지 않는다.
- 내용을 전부 펼친 뒤, 자연스럽게 몇 장이 필요한지 결정한다.

즉 `한 HTML`은 파일 관리 방식일 뿐이다.
실제로는 그 안에 직무별 포트폴리오 3개가 들어간다.

## 2. 절대 건드리지 않을 부분

### 에이전트 개발자용 포트폴리오

현재 `portfolio-v2/index.html`에 만들어진 에이전트 중심 포트폴리오는 이미 완성된 자산으로 본다.

따라서 원칙은 아래다.

- 기존 에이전트 포트폴리오의 흐름은 건드리지 않는다.
- 기존 에이전트 포트폴리오를 줄이지 않는다.
- 기존 에이전트 포트폴리오를 백엔드/솔루션 내용에 맞춰 섞지 않는다.
- 한 HTML 안에서 앞쪽 구간으로 그대로 둔다.

현재 에이전트 포트폴리오에는 이미 아래 내용이 들어가 있다.

- 표지
- 프로젝트 맵
- `HR Copilot` 개요
- `HR Copilot` 문제정의와 구조
- LangSmith Trace 기반 분석
- 품질 평가와 개선 결과
- `BIZ-FUND-AI`의 RAG, 비용, 데이터 운영 일부
- 클로징

이 부분은 “기존 완성본”으로 취급한다.
새로 기획할 대상은 주로 `백엔드용 포트폴리오`와 `솔루션엔지니어용 포트폴리오`다.

## 3. 전체 HTML 배치 방식

HTML은 하나만 둔다.
다만 내부 구조는 이렇게 간다.

- 에이전트 개발자용 포트폴리오: 기존 완성본 유지
- 백엔드 개발자용 포트폴리오: 기존 뒤에 새로 추가
- 솔루션엔지니어용 포트폴리오: 백엔드 뒤에 새로 추가

PDF 제출 방식:

- 에이전트 지원 시: 에이전트 구간만 출력
- 백엔드 지원 시: 백엔드 구간만 출력
- 솔루션엔지니어 지원 시: 솔루션 구간만 출력

따라서 전체 HTML이 길어지는 것은 문제가 아니다.
오히려 각 구간이 직무별 제출본으로 충분히 완성되어야 한다.

## 4. 페이지 수를 정하는 방식

페이지 수를 먼저 정하지 않는다.

작업 순서는 아래처럼 간다.

1. `BIZ-FUND-AI`에서 백엔드용으로 쓸 수 있는 내용을 전부 뽑는다.
2. `BIZ-FUND-AI`에서 솔루션엔지니어용으로 쓸 수 있는 내용을 전부 뽑는다.
3. 비슷한 내용끼리 묶는다.
4. 한 묶음이 한 페이지로 충분하면 한 페이지로 만든다.
5. 내용이 많으면 자연스럽게 두 페이지로 나눈다.
6. 그렇게 나온 결과가 최종 페이지 수다.

즉 페이지 수는 결과다.
출발점이 아니다.

## 5. 백엔드 개발자용 포트폴리오에 반드시 꺼낼 내용

백엔드 지원용 포트폴리오는 `BIZ-FUND-AI`를 메인 프로젝트로 가져간다.
이 프로젝트는 직접 기획, 설계, 구현한 프로젝트이기 때문에 백엔드 포트폴리오의 중심으로 쓰기에 충분하다.

아래 항목들은 전부 페이지 후보로 본다.

## 5-1. 백엔드용 표지

페이지 의도:

- 백엔드 지원용 포트폴리오가 여기서 시작된다는 것을 분명히 보여준다.

써야 할 내용:

- `BIZ-FUND-AI`가 메인 프로젝트라는 점
- 직접 기획, 설계, 구현했다는 점
- FastAPI, PostgreSQL, pgvector, Elasticsearch, 배치, 관리자 기능까지 다뤘다는 점

메시지 후보:

- `정책자금 추천 서비스를 기획부터 운영 구조까지 설계한 백엔드 개발자`
- `AI 기능을 서비스 백엔드 안에서 안정적으로 작동하게 만든 프로젝트`

## 8. 실제 HTML 구현 기준

구현할 때 기준은 이렇다.

- 기존 에이전트 포트폴리오 구간은 유지한다.
- 그 뒤에 백엔드 포트폴리오 구간을 새로 붙인다.
- 그 뒤에 솔루션엔지니어 포트폴리오 구간을 새로 붙인다.
- 페이지 번호는 전체 HTML 기준으로 이어져도 된다.
- 대신 각 구간의 표지와 라벨은 명확히 다르게 둔다.

구간 라벨 예시:

- `AI Agent Developer Portfolio`
- `Backend Developer Portfolio`
- `Solution Engineer Portfolio`

중요한 점:

- 백엔드 구간을 만들 때 에이전트 포폴을 줄이지 않는다.
- 솔루션 구간을 만들 때 백엔드 포폴을 줄이지 않는다.
- 전체 장수가 늘어나는 것은 정상이다.

## 9. 다음 작업 순서

다음 실제 작업 순서는 아래가 맞다.

1. 현재 `index.html`의 에이전트 포트폴리오를 보존 구간으로 표시한다.
2. `BIZ-FUND-AI` 백엔드용 페이지 후보를 위 목록 기준으로 HTML 페이지로 하나씩 만든다.
3. 실제 작성하면서 한 페이지에 들어갈 분량이면 합치고, 넘치면 나눈다.
4. 백엔드 구간이 끝난 뒤 솔루션엔지니어 구간을 같은 방식으로 만든다.
5. 마지막에 PDF 출력용 페이지 범위 안내를 따로 정리한다.

이 계획의 핵심은 한 줄이다.

`페이지 수를 정해서 내용을 줄이는 게 아니라, 쓸 수 있는 내용을 전부 꺼낸 뒤 페이지가 따라오게 만든다.`
