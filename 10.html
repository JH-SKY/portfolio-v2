05

왜 Hybrid RAG가 필요했는지,
그리고 왜 Router로 노드를 분리했는지를 설명할 수 있어야 했습니다.

Why Hybrid RAG

정책자금 질문은 표현이 너무 다양했습니다.

“초기 창업 지원”, “창업 자금”, “사업 시작할 때 받을 수 있는 지원”처럼 같은 의도를 서로 다른 표현으로 말합니다. 키워드만으로는 놓치고, 벡터만으로는 조건 차이를 흐릴 수 있어 두 방식을 결합했습니다.

Keyword Search: 조건어와 제도명을 놓치지 않음
Vector Search: 생활어 표현과 의미 유사성 보완
RRF: 두 결과를 합쳐 상위 문서 안정화
Why Router

사용자 질문은 한 가지 유형으로만 끝나지 않았습니다.

어떤 질문은 검색이 필요하고, 어떤 질문은 조건 진단이 필요하며, 어떤 질문은 통계나 시뮬레이션까지 필요했습니다. 하나의 노드에 다 몰아넣으면 이유도, 실패 지점도 설명하기 어려워졌습니다.

Design Principle

Troubleshooting | Operating Reliability

이 프로젝트의 강점은 기능 설명이 아니라,
실제 운영 장애를 복구한 기록입니다.

Issue 01

순환 참조

Soft Delete 전환 과정에서 서비스 간 참조가 꼬이면서 AuthService와 BusinessService가 서로 물고 있었습니다.

Method Injection + Router Orchestration
서비스 직접 참조를 끊고 조합 책임을 Router로 올렸습니다.
Issue 02

Rollback 중 데이터 유실

중복 예외가 발생했을 때 이전 성공 데이터까지 함께 롤백되는 문제가 있었습니다.

UPSERT + 트랜잭션 범위 재설계
1,282건을 유실 없이 처리하도록 배치 흐름을 수정했습니다.
Issue 03

파일 파싱 실패가 전체 롤백으로 확산

확장자와 실제 바이너리가 다를 때 일부 실패가 전체 배치를 망가뜨렸습니다.

Magic Number + Savepoint
파일 형식을 실제 바이트로 확인하고 실패 범위를 국소화했습니다.
Issue 04

Admin API 500 / 404

파라미터명이 바뀌었지만 호출부 반영이 누락되어 500이 발생했고, 연동되지 않은 API는 404가 반복됐습니다.

응답 계약 정리 + 기본 응답 도입
화면 장애를 막고 Feedback / Monitoring / Insights 흐름을 안정화했습니다.
Issue 05

Supabase Timeout + AI 요약 누락

Full Sync에서 커넥션이 끊기고, Swagger 기본값 때문에 `ai_summary`가 NULL로 저장되는 문제가 겹쳤습니다.

pool 설정 조정 + 페이지 단위 커밋
timeout을 줄이고 오류 유형을 분리해 1,255건 누락 문제를 복구했습니다.
