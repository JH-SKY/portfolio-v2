Project 2 BIZ-FUND-AI

AI 기능만 만든 것이 아닙니다. 운영에서 터지는 데이터·배치·트랜잭션 문제를 구조적으로 해결했습니다.

Issue 01 · 데이터 유실

로그 성공 1,282건,
실제 DB엔 300건

원인rollback이 이전 성공분까지 전량 되돌림
핵심 깨달음트랜잭션 원자성 오해
해결 UPSERT + 트랜잭션 범위 재설계
결과: 1,282건 유실 없이 저장 달성
Issue 02 · 파일 파싱 실패

파일명은 .pdf
실제 형식은 HWP

원인확장자 기반 판단 + 전체 배치 롤백
핵심 깨달음외부 API 데이터는 신뢰 불가
해결 Magic Number + Savepoint (중첩 트랜잭션)
결과: 파싱 성공률 24% → 57%
Issue 03 · 배치 세션 끊김

200건 이후
DB 연결이 끊김

원인Supabase pgbouncer 5분 idle timeout
핵심 깨달음배치 전체를 단일 장수 세션으로 처리
해결 100건/페이지 fresh session 구조
결과: 1,255건 Full Sync 안정화
Working Method

03. 구조 수정 노드 책임, 검색 방식, 트랜잭션 경계를 다시 설계한다.
04. 재측정 동일 기준으로 다시 비교해 개선 이유를 설명한다.
