Data Pipeline Reliability04 / 12

Batch failure isolation

정책 수집/구조화/임베딩 배치에서 중복 실행과 부분 실패를 각각 다른 방식으로 막았습니다.

SchedulerAPScheduler가 매일 정책 동기화 job을 실행
Advisory Lockpg_try_advisory_lock으로 leader instance만 실행
UPSERT중복 정책은 ON CONFLICT DO UPDATE로 반영
Savepoint개별 정책 실패를 begin_nested로 격리
Embedding변경된 정책만 임베딩하고 batch 처리

운영 리스크

서버 인스턴스가 늘면 같은 배치가 여러 번 실행될 수 있고, 한 정책 처리 실패가 전체 transaction을 오염시킬 수 있습니다.

설계 판단

중복 실행은 DB advisory lock으로, 부분 실패는 savepoint로, 중복 데이터는 UPSERT로 각각 분리해서 해결했습니다.

Supabase pgbouncer

문제

Hybrid RAG06 / 12

Search quality

정책 문서의 구조를 살리는 chunking과 고유명사 검색을 보완하는 hybrid retrieval을 설계했습니다.

Chunk

섹션 기반 분리

지원대상, 지원내용, 신청방법 같은 정책 문서 헤더를 기준으로 chunk를 만듭니다.

Fallback

Sliding window

섹션이 없을 때 800자 window와 100자 overlap으로 fallback합니다.

Context

Embedding prefix

정책명, 기관, 지원유형을 embedding 입력에 붙이고 DB에는 원문 chunk를 저장합니다.

Cost

Content hash

SHA-256 hash가 같으면 재임베딩을 skip해 API 비용과 DB write를 줄입니다.

검색 문제구현 판단검증 근거

의미 유사 질문pgvector cosine searchdense retrieval label
두 결과의 순위 통합RRF 기반 hybrid resultsparse ranking test
