Why Hybrid RAG

“창업 지원금”과
“초기창업패키지”를
같은 질문으로 찾아야 했습니다.

Keyword Search Vector Search RRF Re-ranking

단일 검색은 표현 차이에 약했습니다. 그래서 키워드 검색과 의미 검색을 결합해 관련 문서를 안정적으로 끌어올렸습니다.

Why Node Separation

질문 하나에 검색, 진단,
통계, 시뮬레이션이 함께 필요했습니다.

Router의도 판단
Diagnosis조건 분석
RAG근거 검색

Simulation가능성 계산

하나의 LLM에 몰아주면 추적과 복구가 어려워서, 역할을 분리해 설계 이유를 설명 가능한 구조로 만들었습니다.

Issue 01

200~300건 이후 멈춤

실제 형식이 다름

Working Method

프로젝트가 달라도,
문제 푸는 방식은 같습니다.

HR Copilot에서는 품질 문제를 추적했고, BIZ-FUND-AI에서는 설계와 운영 장애를 풀었습니다. 공통점은 언제나 현상을 고정하고 구조를 바꾼다는 점입니다.

Method
01. 현상 고정점수, Trace, DB 건수, 에러 메시지로 문제를 보이게 만든다.
02. 가설 분리원인을 섞지 않고 하나씩 나눠 본다.
03. 구조 수정노드 책임, 검색 방식, 트랜잭션 경계를 다시 설계한다.
04. 재측정동일 기준으로 다시 비교해 개선 이유를 설명한다.
